# Sourcing Request Page

## Metadata

- Route: dedicated sourcing route
- Template type: conversion page
- Primary audiences: buyers with non-standard or hard-to-source needs
- Primary CTA: Submit Sourcing Request
- Secondary CTA: Talk To Trade Desk

## Goal

Support complex sourcing inquiries that need more context than a standard quote form.

## Section Plan

### 1. Hero
- Purpose: Explain when to use this route instead of a standard quote.
- Content: use-case examples and scope explanation.

### 2. Use Cases
- Purpose: Help users self-select correctly.
- Content: multi-origin demand, unusual specs, route-sensitive sourcing, urgent supply gaps.

### 3. Detailed Sourcing Form
- Purpose: Capture problem context.
- Content: target product, acceptable alternatives, quantity, destination, timeline, constraints, documents, attachments.

### 4. Trade Desk Support Strip
- Purpose: Reinforce human support.
- Content: how trade desk follows up and what information helps most.

### 5. FAQ
- Purpose: Reduce uncertainty.
- Content: response expectations, confidentiality, attachment handling, fallback options.

## State Coverage

- Default: extended sourcing form.
- Loading: upload and form-control skeletons.
- Empty: not applicable.
- Success: request accepted with follow-up expectations.
- Error: preserve long-form entry and upload metadata on retry where possible.
