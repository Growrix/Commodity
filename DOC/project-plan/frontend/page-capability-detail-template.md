# Capability Detail Template Page

## Metadata

- Route: `/capabilities/[slug]`
- Template type: dynamic CMS template
- Primary audiences: buyers, partners
- Primary CTA: Talk To Our Team
- Secondary CTA: Explore Related Markets

## Goal

Explain one capability deeply enough that a visitor understands process, value, proof, and next action.

## Section Plan

### 1. Capability Hero
- Purpose: Define the capability and why it matters.
- Content: summary, promise, sector relevance.

### 2. Scope Of Service
- Purpose: Clarify what is included.
- Content: process steps, ownership boundaries, typical deliverables.

### 3. Proof And Controls
- Purpose: Show operational rigor.
- Content: QA controls, documentation steps, commercial safeguards, metrics.

### 4. Use Cases
- Purpose: Make the capability concrete.
- Content: buyer scenarios, supplier scenarios, corridor relevance.

### 5. Related Content
- Purpose: Route deeper into sectors or insights.
- Content: linked markets, case studies, related products.

### 6. CTA Block
- Purpose: Capture interest.
- Content: discuss capability, request quote, partnership CTA.

## State Coverage

- Default: detailed capability narrative.
- Loading: hero and proof-card skeletons.
- Empty: capability summary with generic CTA.
- Success: not applicable.
- Error: safe fallback to capabilities hub.
