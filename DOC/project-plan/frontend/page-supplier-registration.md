# Supplier Registration Page

## Metadata

- Route: `/partner-with-us/suppliers`
- Template type: conversion page
- Primary audiences: suppliers and origin-side partners
- Primary CTA: Submit Supplier Registration
- Secondary CTA: Contact Supplier Review Team

## Goal

Set clear supplier expectations and collect the documentation and profile information needed for structured review.

## Section Plan

### 1. Hero
- Purpose: Frame supplier onboarding clearly.
- Content: supplier value proposition, markets of interest, response expectations.

### 2. What We Look For
- Purpose: Self-qualify submissions.
- Content: product categories, regions, certifications, export readiness.

### 3. Required Documents And Standards
- Purpose: Prevent incomplete submissions.
- Content: company profile, certifications, origin documents, QA references.

### 4. Supplier Form
- Purpose: Collect structured supplier data.
- Content: company, contact, commodities, capacity, origins, certifications, file uploads.

### 5. Review Timeline And Privacy Note
- Purpose: Clarify what happens after submission.
- Content: review steps, confidentiality, possible follow-up requests.

### 6. FAQ
- Purpose: Reduce friction and support completion.
- Content: file types, approval timelines, document handling, next steps.

## State Coverage

- Default: supplier onboarding page.
- Loading: upload-zone and form-control skeletons.
- Empty: not applicable.
- Success: structured confirmation with reference ID if available.
- Error: invalid file guidance, retry-safe upload failures, inline validation.
