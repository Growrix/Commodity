# Request A Quote Page

## Metadata

- Route: `/partner-with-us/buyers` or dedicated quote route
- Template type: conversion page
- Primary audiences: qualified buyers
- Primary CTA: Submit Quote Request
- Secondary CTA: Explore Products

## Goal

Capture high-intent buying demand with enough detail for commercial qualification while keeping the flow low-friction.

## Section Plan

### 1. Hero
- Purpose: Clarify what the quote process is for.
- Content: value proposition, expected response time, confidentiality note.

### 2. How The Process Works
- Purpose: Build confidence before form completion.
- Content: short step sequence from submission to team response.

### 3. Qualification Form
- Purpose: Collect actionable commercial detail.
- Content: product, grade, volume, destination, timeline, company info, optional notes.

### 4. Supporting Proof
- Purpose: Reduce hesitation.
- Content: trust badges, documentation support, market coverage.

### 5. FAQ
- Purpose: Address common buying concerns.
- Content: response time, MOQ guidance, documentation, origin flexibility.

### 6. Alternate Contact Block
- Purpose: Preserve conversion if form completion fails.
- Content: regional email, phone, or general contact route.

## State Coverage

- Default: quote form page.
- Loading: form field skeletons if schema or options are fetched.
- Empty: not applicable.
- Success: inline thank-you state with next-step expectations.
- Error: validation errors inline; submission failure preserves form values and shows retry-safe message.
