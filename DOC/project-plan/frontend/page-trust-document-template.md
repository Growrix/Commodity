# Trust Document Template Page

## Metadata

- Route: `/trust-center/[slug]`
- Template type: policy or trust-document template
- Primary audiences: buyers, suppliers, compliance reviewers
- Primary CTA: Contact Our Team
- Secondary CTA: Return To Trust Center

## Goal

Present policy, certification, or trust content in a clear, reviewable, low-friction format.

## Section Plan

### 1. Document Header
- Purpose: Identify the document clearly.
- Content: title, document type, effective date, owner, summary.

### 2. Body Content
- Purpose: Present the policy or trust material cleanly.
- Content: sectioned rich text, downloadable asset link if relevant.

### 3. Review Metadata
- Purpose: Improve trust and accountability.
- Content: last reviewed date, owner, related policy links.

### 4. Related Documents
- Purpose: Support due-diligence exploration.
- Content: related trust docs, FAQ links, contact path.

### 5. CTA Footer
- Purpose: Offer follow-up path.
- Content: contact compliance, contact sales, return to hub.

## State Coverage

- Default: readable trust document.
- Loading: heading and body skeletons.
- Empty: unavailable document fallback with hub link.
- Success: not applicable.
- Error: safe fallback and contact CTA.
